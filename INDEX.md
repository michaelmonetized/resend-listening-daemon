## 🚀 Getting Started

### 1. **Install**
```bash
cd ~/Projects/resend-listening-daemon
bash scripts/install.sh
```

### 2. **Configure**
```bash
nano ~/.config/resendld/boxes.json
# Add your Resend email boxes
```

### 3. **Start**
```bash
resendld start
# Opens daemon + Convex + web server
```

### 4. **Use**
```bash
# Check status
resendld status

# View logs
resendld logs

# Manage boxes
resendld box list
resendld box add user@example.com
```

### 5. **Access**
Open https://resendld.localhost in your browser

## 📝 File Descriptions

### Daemon (Listening + Storage)

**listen.ts** (C1, 180 lines)
- Spawns `resend emails receiving listen --to <email>`
- Parses JSON + SMTP email formats
- Generates messageId
- Routes to storage + gateway + Convex

**gateway.ts** (C2, 140 lines)
- POSTs email events to OpenClaw gateway
- Exponential backoff retry logic
- Formats structured `email.received` events
- Handles network timeouts

**storage.ts** (CC1, 200+ lines)
- Writes emails as markdown files
- Directory structure: `~/.openclaw/workspace/mail/inbox/{from}/{subject-date-id}/`
- Creates MSG.md with YAML frontmatter
- Downloads attachments to `attachments/` folder
- Generates index.json for quick lookup

### Convex Backend

**schema.ts** (C3, 90 lines)
- Defines `messages`, `attachments`, `boxes` tables
- Indexes for efficient queries (messageId, from, boxEmail, date)
- TypeScript types (Message, Attachment, Box)

**messages.ts** (C4, 290 lines)
- `getMessages()` — Query inbox with pagination + search
- `getMessage()` — Fetch message with attachments
- `storeMessage()` — Insert new message
- `markAsRead()`, `toggleStar()`, `addLabel()`, `archiveMessage()`, `markAsSpam()`, `deleteMessage()`

**attachments.ts** (CC9, 50 lines)
- `getAttachments()` — List attachments for message
- `storeAttachmentMetadata()` — Save attachment info
- `deleteAttachments()` — Cascade delete on message delete

**boxes.ts** (100 lines)
- `getBoxes()`, `getBox()` — Query boxes
- `addBox()`, `updateBox()`, `deleteBox()` — Manage boxes
- Archives messages instead of hard delete

### Web Frontend

## 📚 Learning Resources

### Understanding the Code
1. Start with **PROJECT_PLAN.md** for the big picture
2. Read **IMPLEMENTATION_NOTES.md** for technical details
3. Look at **src/daemon/listen.ts** to see email flow
4. Explore **web/convex/schema.ts** to understand database design
5. Check **web/src/routes/index.tsx** to see how UI connects to data

### Extending the Project
1. Add new query in **web/convex/messages.ts**
2. Use it in **web/src/components/** or **web/src/routes/**
3. Style with Tailwind classes (see existing components)
4. Test with `bun run dev` (web/) + manual browser testing

### Debugging
- **Daemon logs:** `resendld logs`
- **Web console:** Browser DevTools (F12)
- **Convex logs:** `npx convex dev` output (terminal)
- **TypeScript errors:** `cd web && bun run type-check`
